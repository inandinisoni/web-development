ABOUT CSS CASCADE AND SPECIFICITY

CSS stands for Cascade style sheet . Cascade is an algorithm for solving conflicts where multiple CSS rules are applied to an HTML element

Specificity is the means by which browsers decide which CSS property values are the most relevant to an element and therefore will be applied.

Specificity is a weight that is applied to a given CSS declaration, determined by the number of each selector type in the matching selector. When multiple declarations have equal specificity, the last declaration found in the CSS is applied to the element.

Specificity Hierarchy

Inline styles - 1000

ID - 100

Class, pseudo-class, attribute - 10

Element, pseudo-element - 1

Universal selector - 0

Inline CSS > ID selector > Class or attribute selector > Element selector > Universal Selector

!important

It is important to note that the !important declaration will override the normal specificity hierarchy.

For example, if you have a style defined for a paragraph element with an ID of #special, and you add the !important declaration to the style that is defined for the .special class, the style that is defined for the .special class will take precedence over the style that is defined for the ID.

It is generally best to avoid using the !important declaration, as it can make it difficult to override styles later on in your CSS.

Position and order of apppearance

When two selectors have the same specificity, the last one defined will take precedence.
